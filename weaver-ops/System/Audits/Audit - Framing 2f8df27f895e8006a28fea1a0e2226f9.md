# Audit - Framing

### 1. Context

- Client name: Maxime
- Project name: Qreno
- Date: 5 january 2026
- People involved: CS (Corolla), PM (Eva), Designer UX (Allan), Designer UI (Lise)

---

### 2. Trigger

- What triggered the start of the Framing? I send the client informations to Corolla
- Who initiated it? Corolla
- Tool used: Whatsapp

---

### 3. Chronological Steps (what actually happened)

**Step 1**

- Action performed: Welcome and reassurance call
- Who did it: Corolla
- Tool: Phone
- Output produced: First call of onboarding booked

**Step 2**

- Action performed: Onboarding call
- Who did it: Corolla
- Tool: Google meet
- Output produced: Client get the roadmap and the first session with the PM is setup, whatsapp group created with all the people (Client, Corolla, Jalil, Eva, Allan, Lise)

**Step 3**

- Action performed: Session with the PM
- Who did it: Eva
- Tool: Google meet
- Output produced: gathering of informations regarding the project and the need
- Conditions: We can do multiples sessions until we gather all the required informations

**Step 4**

- Action performed: Formalize the brief for the designer UX
- Who did it: Eva
- Tool: Google docs
- Output produced: Complete and detailed brief to design the UX

**Step 5**

- Action performed: Send the brief to the client by mail and ask a confirmation before handover it to the designer
- Who did it: Eva
- Tool: Gmail
- Output produced: Confirmation or adjustments required by the client
- Conditions: We can do multiples adjustments until we have the confirmation of the client

**Step 6**

- Action performed: Handover the brief to the UX designer
- Who did it: Eva
- Tool: Discord
- Output produced: The UX designer get all the informations

**Step 7**

- Action performed: Design of the UX
- Who did it: Allan
- Tool: Figma
- Output produced: UX design animated and components system

**Step 8**

- Action performed: UX designer give us the link of Figma
- Who did it: Allan
- Tool: Discord
- Output produced: We can setup the next call with the client

**Step 9**

- Action performed: Organize the review with the client
- Who did it: Eva
- Tool: Whatsapp
- Output produced: Call setup

**Step 10**

- Action performed: Review with the client
- Who did it: Eva
- Tool: Google meet
- Output produced: Confirmation of the client or gather adjustments from the client
- Conditions: We can  go again to the step 6 until we don’t have the confirmation of the client

**Step 11**

- Action performed: Handover to the UI designer
- Who did it: Eva
- Tool: Discord
- Output produced: UI designer get the UX design

**Step 12**

- Action performed: UI design of the UX design
- Who did it: Lise
- Tool: Figma
- Output produced: UI design animated

**Step 13**

- Action performed: Inform the client that UI is improved
- Who did it: Corolla
- Tool: Call & Whatsapp
- Output produced: Client see the new version of the Figma

**Step 14**

- Action performed: Confirmation or adjustments required by the client
- Who did it: Client
- Tool: Whatsapp
- Output produced: Lise adjust the design
- Conditions: We can go to step 13 until we don’t have the client confirmation

**Step 15**

- Action performed: Finalization of the product documentation
- Who did it: Eva
- Tool: Google docs
- Output produced: Complete and detailed product documentation

### 4. Validations & Decisions

- What was validated? Design & Documentation
- By whom? The client

    - Lack of preparation before the meetings (no client and projet informations for the PM before she discover the client)
    - Creation of documentation not standardized
- Where did it block? Nowhere
- Why? For Nothing
