# State 2 - Development Execution

### **Goal**

Execute development to deliver the validated product scope with controlled quality and no remaining critical risk.

---

## **State Owner**

Tech Lead

---

## **Required Functions**

- Tech Lead
- Backend Developer
- Frontend Developer
- Mobile Developer
- QA / Tester
- Project / Product Lead

---

**Entry conditions**

- State 1 closed
- Work organization ready
- Delivery resources allocated or recruitment in progress
- Repositories, tooling, infrastructure, and accesses ready

**Exit conditions**

- All planned features developed
- No blocking or critical bugs remain
- Product ready for final validation

---

## Actions

**Action 1 – Execute Development Work**

R: Backend Developer / Frontend Developer / Mobile Developer · A: Tech Lead

Purpose: implement all planned features according to validated scope and design.

Must be true after: all features implemented; code merged according to defined workflow rules.

---

**Action 2 – Perform Continuous Quality Control**

R: QA / Tester · A: Tech Lead

Purpose: detect and correct defects during development.

Must be true after: no blocking or critical bugs remain; quality level acceptable.

---

**Action 3 – Manage Scope Adjustments**

R: Project / Product Lead · A: Project / Product Lead

Purpose: handle any scope change without destabilizing execution.

Must be true after: scope change rejected or formally documented and approved.
