# State 2 - Assignment Confirmation

## Goal

Formally assign, onboard, and lock the project team.

---

## State Owner

Staffing / Capacity Manager

---

## Required Functions

- Staffing / Capacity Manager
- Project / Product Lead

**Entry conditions**

- Team selection validated
- Developers available to start

**Exit conditions**

- Developers formally assigned
- Developers onboarded and locked to the project

---

## Actions

**Action 1 – Communicate Assignment**

R: Staffing / Capacity Manager · A: Staffing / Capacity Manager

Purpose: formally notify developers of their assignment.

Must be true after:

Developers acknowledge assignment to the project.

---

**Action 2 – Onboard Developers to Project**

R: Project / Product Lead · A: Project / Product Lead

Purpose: integrate developers into the project context.

Must be true after:

Project brief shared, accesses granted, and initial tasks assigned.

---

**Action 3 – Lock Allocation**

R: Staffing / Capacity Manager · A: Staffing / Capacity Manager

Purpose: make the assignment irreversible without a new workflow.

Must be true after:

Developers are considered unavailable for reassignment unless a new allocation workflow is executed.
