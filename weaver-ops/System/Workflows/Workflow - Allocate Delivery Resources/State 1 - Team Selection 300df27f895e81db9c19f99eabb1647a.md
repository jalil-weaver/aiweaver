# State 1 - Team Selection

## Goal

Select a concrete project team and obtain explicit technical validation.

---

## State Owner

Staffing / Capacity Manager

---

## Required Functions

- Staffing / Capacity Manager
- Tech Lead

---

**Entry conditions**

- Availability Assessment completed
- Required resources confirmed available

**Exit conditions**

- Project team selected
- Technical validation explicitly accepted

---

### **Actions**

**Action 1 – Propose Team Allocation**

R: Staffing / Capacity Manager · A: Staffing / Capacity Manager

Purpose: select a concrete team from the available developer list.

Must be true after:

One proposed team exists with named developers, roles, allocation percentages, and start dates.

---

**Action 2 – Validate Technical Fit**

R: Tech Lead · A: Tech Lead

Purpose: verify that the proposed team meets technical and project requirements.

Must be true after:

Team proposal is explicitly accepted or rejected with documented reasons.

---

**Action 3 – Adjust Selection if Rejected**

R: Staffing / Capacity Manager · A: Staffing / Capacity Manager

Purpose: produce a revised proposal if validation fails.

Must be true after:

A technically validated team proposal exists.
