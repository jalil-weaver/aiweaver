# State 0 - Estimation Readiness Check

## State Owner

Delivery Owner

---

## Required Functions

- Delivery Owner
- Project / Product Lead
- Client Interface (Delivery Side)

---

**Entry Conditions**

- Inputs received (A, B, or C)

---

**Exit Conditions**

- Inputs declared sufficient
    
    **or**
    
- Estimation blocked with missing inputs documented

---

### **Actions**

**Action 1 — Review Inputs Sufficiency**

R: Delivery Owner · A: Delivery Owner

Purpose: determine whether estimation can start without assumptions.

Result: inputs classified as sufficient or insufficient.

**Action 2 — Produce Estimation Feedback (if insufficient)**

R: Client Interface (Delivery Side) · A: Delivery Owner

Purpose: document precisely what is missing to estimate.

Result: structured feedback sent to Sales.
