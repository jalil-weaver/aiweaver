# State 3 - Technical Definition

### **Goal**

Define **how the product will be built**, based on the validated product scope and design, with no remaining technical ambiguity.

---

## **State Owner**

Tech Lead

---

## **Required Functions**

- Tech Lead
- Project / Product Lead
- Staffing / Capacity Manager

---

### **Entry Conditions**

- State 2 closed
- Design validated as a whole

---

### **Exit Conditions**

- Technical Documentation completed
- Technical choices validated
- Infrastructure and environment assumptions defined
- **Technical Documentation validated as a whole**
- Delivery Team produced

---

## **Actions**

**Action 1 – Define Technical Architecture**

R: Tech Lead · A: Tech Lead

Purpose: establish the overall technical structure of the product.

Must be true after: architecture clearly defined; responsibilities between components explicit.

---

**Action 2 – Define Technical Stack and Key Decisions**

R: Tech Lead · A: Tech Lead

Purpose: remove uncertainty on technologies and major implementation choices.

Must be true after: stack selected; key technical decisions documented and justified.

---

**Action 3 – Define Data Models and Integrations**

R: Tech Lead · A: Tech Lead

Purpose: clarify how data is structured and how external systems are connected.

Must be true after: data models defined; integrations identified and specified.

---

**Action 4 – Define Infrastructure and Environment Assumptions**

R: Tech Lead · A: Tech Lead

Purpose: define hosting, environments, and operational assumptions.

Must be true after: infrastructure model and environments clearly defined.

---

**Action 5 – Validate Technical Definition**

R: Tech Lead · A: Tech Lead

Purpose: formally validate the technical documentation before delivery planning.

Must be true after: no open technical questions remain; documentation validated.

---

**Action 6 – Define Delivery Team**

R: Staffing / Capacity Manager · A: Staffing / Capacity Manager

Purpose: define the minimal delivery capacity required to operate the product in continuous delivery.

Must be true after: required functions identified; seniority defined; required technologies and methodologies specified; minimal run capacity established.
