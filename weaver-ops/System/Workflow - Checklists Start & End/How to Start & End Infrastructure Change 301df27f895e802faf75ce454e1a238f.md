# How to Start & End Infrastructure Change

- ☐ Infrastructure Change Brief exists (Yes / No)
- ☐ Infrastructure Change Brief is created by a Tech Lead (Yes / No)
- ☐ Infrastructure Change Brief is readable and complete (Yes / No)
- ☐ Project is currently in **Ongoing Delivery** or **Maintenance / Continuous Delivery** state (Yes / No)
- ☐ Relevant technical documentation exists (Yes / No)
- ☐ Delivery scenario exists if applicable (Yes / No)
- ☐ Access to current infrastructure is available (Yes / No)
- ☐ No infrastructure execution has started yet (Yes / No)

---

# **How to End Infrastructure Change**

### Decision Completion

- ☐ Infrastructure Change Proposal exists (Yes / No)
- ☐ Proposal includes a clearly defined target solution (Yes / No)
- ☐ Proposal includes cost estimation (Yes / No)
- ☐ Proposal includes risk assessment (Yes / No)

### Explicit Decision

- ☐ Proposal decision is explicitly recorded (Yes / No)
- ☐ Decision status is one of:
    - ☐ Approved
    - ☐ Rejected
    - ☐ Needs adjustment

### Routing Integrity

- ☐ If **Approved**, DevOps is formally informed (Yes / No)
- ☐ If **Approved**, DevOps execution is authorized **only based on the approved proposal** (Yes / No)
- ☐ If **Rejected**, no infrastructure work is allowed (Yes / No)
- ☐ If **Needs adjustment**, proposal is updated and routed back for review (Yes / No)

Checklist

### **State 0 – Infrastructure Change Qualification**

☐ Infrastructure Change Brief reviewed

☐ Infra impact is clearly identified or ruled out

☐ Request is classified as **infra-related** or **not infra-related**

---

### **State 1 – Infrastructure Impact Analysis**

☐ Current infrastructure constraints and risks identified

☐ Required infrastructure changes clearly listed

☐ Infra impacts are sufficiently understood to design a solution

---

### **State 2 – Infrastructure Solution & Estimation**

☐ Target infrastructure solution defined

☐ Cost ranges estimated

☐ Infrastructure risks identified

☐ Infrastructure Change Proposal completed

---

### **State 3 – Decision & Routing**

☐ Infrastructure proposal technically reviewed

☐ Decision made: approved, rejected, or needs adjustment

☐ Outcome routed accordingly (execution, stop, or revision)
