# How to Start & End Continuous Delivery

### Roadmap & Scope

- ☐ Product Roadmap exists (Yes / No)
- ☐ Product Roadmap is complete (multi-month view) (Yes / No)
- ☐ Product Roadmap is **not frozen** (Yes / No)

---

### Documentation (Blocking)

- ☐ Product Documentation exists (Yes / No)
- ☐ Product Documentation is exploitable for delivery (Yes / No)
- ☐ Technical Documentation exists (Yes / No)
- ☐ Technical Documentation is exploitable for delivery (Yes / No)
- ☐ Design exists (Yes / No)
- ☐ Design is exploitable for delivery (Yes / No)

---

### Team & Ownership

- ☐ Delivery Team is defined (Yes / No)
- ☐ Delivery Team includes at least **1 developer** (Yes / No)
- ☐ Weaver PM is assigned (Yes / No)
- ☐ PM back-up is identified (Yes / No)

---

### Access & Tooling

- ☐ Minimum required accesses are available (Yes / No)
- ☐ If accesses are missing, a decision to standardize tools and infrastructure is explicitly recorded (Yes / No)

---

# **How to End – Workflow: Continuous Delivery**

### Capacity Installation

- ☐ Minimal delivery capacity is installed (Yes / No)
- ☐ Delivery capacity can be activated immediately (Yes / No)

---

### Environments

- ☐ Test environment is operational (Yes / No)
- ☐ Production environment is operational (Yes / No)

---

### Tooling & Repositories

- ☐ Required repositories are created and accessible (Yes / No)
- ☐ Required tools are configured and accessible (Yes / No)

---

### Execution Readiness

- ☐ Kanban board is active (Yes / No)
- ☐ Backlog is exploitable (prioritized, actionable items exist) (Yes / No)
- ☐ No remaining installation task blocks execution (Yes / No)

Checklist

### **State 0 – Ready to Start**

☐ Product Roadmap is exploitable for continuous delivery

☐ Product, Technical Documentation, and Design are validated

☐ Delivery Team and PM (with back-up) are confirmed

☐ No blocking ambiguity remains to start installation

---

### **State 1 – Delivery Design**

☐ Global backlog is created and exploitable

☐ Delivery execution rules are defined

☐ Staffing process is authorized

☐ Delivery model is ready to be installed

---

### **State 2 – Capacity Installation**

☐ Minimal delivery capacity is installed (≥ 1 developer)

☐ Delivery capacity is onboarded on product and rules

☐ Tools, repositories, and environments are operational

---

### **State 3 – Run Activation**

☐ Kanban is active

☐ Backlog is exploitable

☐ Ownership and responsibilities are clear

☐ Delivery run is active
